## Identity

- Built **from Africa, for the world** — never "tech for Africa." Hold the global bar, shipped from here.
- **AI-native**: AI is the substrate, not a feature bolted on. Assume cheap reasoning and abundant agents, and build as if that world is already here.
- **Small and dangerous**: leverage over headcount. Every system we build makes the next one easier.
- **Stewardship**: integrity is the material we build with, excellence is a form of service, and users are ends — never metrics.

## How we build — non-negotiable

- **Build for the constraint.** Assume a low-end phone on a slow, expensive, dropping network. Offline-first, low-bandwidth, resilient. Designing for the hardest case produces better software everywhere — it is our moat, not our handicap.
- **Solve a real, felt problem.** If you cannot name the user and the pain in one sentence, do not build it.
- **Speed and finishing both matter.** Ship fast and improve in public — but "it runs" is not "it's done."
- **Compound, never repeat.** Reuse what exists; never solve the same thing twice. Write what you learn back into the repo / codex so the company gets structurally smarter.
- **Every product is navigable.** Each repo carries a graphify knowledge graph (`graphify-out/`), kept correctly labelled on every commit, so any agent can map the system before it changes it. Code an agent cannot map, it cannot safely change. The graph build is run deliberately (it costs reasoning) — it is never forced automatically.
- **Tell the truth about reality** — what works, what failed, where we are weak — fast and without flinching.
- **Sovereignty.** Keep IP, core data, and leverage with Sampu Dynamics and the people we serve. Protect user data as if it were our own most private information.
- **Dignity.** Meet users in their language, on their device, without condescension.

## The bar — every solution clears three gates

Not two. All three. Fail one and it is a draft, not a finished thing.

### 1. Quality — *Would this hold up in front of the best builders in the world?*
- Works under real conditions (low-end device, 3G/offline), not just the demo machine.
- Fast. Latency is disrespect.
- Reliable; fails gracefully and tells the truth when it fails.
- Secure; protects user data as our own.
- Finished to the edges — error states, empty states, the unglamorous paths.

### 2. Impact — *Does it make a measurable dent for real people?*
- We can name the person served and the problem removed.
- The change is measurable: time, money, friction, access, or a new capability.
- It scales without losing what made it good.

### 3. Value — *Will it be worth more next year than today?*
- Economically durable: people pay, stay, and refer.
- Compounds our moat, data, platform, or reputation — not a dead-end feature.
- Owned and defensible.

## Definition of done

A Sampu Dynamics solution ships only when:

1. It clears all three gates — nothing hand-waved.
2. A stranger can use it under real conditions and succeed **without us in the room**.
3. We would put the Sampu Dynamics name on it in front of the world's best builders.
4. What we learned has been written back into the repo / codex.

Until then it is a draft. Drafts are how we work — we never confuse a draft for the standard.

## Creed

> Built from Africa, for the world.
> AI as foundation, not decoration.
> Built for the hardest conditions — and therefore built robust.
> Real problems, real people, a measurable dent, with dignity.
